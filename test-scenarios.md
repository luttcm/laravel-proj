# Сценарии тестирования

## 1. Цель тестирования

Цель тестирования заключается в проверке корректности работы основных модулей многопользовательской сетевой информационной системы, включая безопасность, ролевую модель, расчеты, отчетность и работу с контентом.

## 3. Примеры детализированных тест-кейсов

### TS-13. Выполнение расчета с валидными данными

Шаги:

1. Авторизоваться под пользователем с ролью `manager`.
2. Открыть раздел расчетов.
3. Ввести корректные параметры сделки.
4. Нажать кнопку запуска расчета.

Ожидаемый результат:

- система выполняет расчет без ошибки;
- на странице отображаются рассчитанные значения.

### TS-18. Создание финансового отчета

Шаги:

1. Авторизоваться под пользователем с ролью `finance`.
2. Открыть раздел финансовых отчетов.
3. Перейти к добавлению новой записи.
4. Заполнить обязательные поля.
5. Сохранить отчет.

Ожидаемый результат:

- запись создается;
- пользователь возвращается к списку отчетов или видит сообщение об успехе.

### TS-22. Создание страницы базы знаний

Шаги:

1. Авторизоваться в системе.
2. Перейти в раздел базы знаний.
3. Открыть форму создания страницы.
4. Заполнить заголовок, контент, категорию и порядок сортировки.
5. При необходимости загрузить изображение.
6. Сохранить страницу.

Ожидаемый результат:

- страница создается;
- автор записи фиксируется системой;
- страница доступна для просмотра в базе знаний.

## 4. Критерии успешного тестирования

Тестирование считается успешным, если:

- критические пользовательские сценарии выполняются без ошибок;
- ограничения ролей соблюдаются;
- данные сохраняются и отображаются корректно;
- система корректно обрабатывает невалидные входные данные;
- функциональность базы знаний, отчетности и расчетов подтверждена практическими сценариями.
