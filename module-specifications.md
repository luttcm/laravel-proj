# Спецификации программных модулей

## 1. Модуль аутентификации и безопасности

Назначение:

- идентификация пользователя;
- открытие и завершение пользовательской сессии;
- двухфакторная аутентификация;
- разграничение прав доступа.

Основные компоненты:

- `AuthController`;
- `AuthService`;
- `TwoFactorController`;
- `CheckRole`;
- `CheckTwoFactor`.

Входные данные:

- email или иной идентификатор пользователя;
- пароль;
- код двухфакторной аутентификации.

Выходные данные:

- установленная пользовательская сессия;
- доступ к разрешенным разделам;
- сообщение об ошибке при неуспешной проверке.

## 2. Модуль управления пользователями

Назначение:

- ведение учетных записей;
- просмотр карточек пользователей;
- редактирование профиля и аватара.

Основные компоненты:

- `UserController`;
- `UserService`;
- `UserRepository`;
- `User`.

Операции:

- создание пользователя;
- редактирование пользователя;
- удаление пользователя;
- просмотр списка и карточки;
- обновление профиля;
- загрузка аватара.

## 3. Модуль новостей и комментариев

Назначение:

- публикация корпоративных новостей;
- обратная связь пользователей через комментарии и лайки.

Основные компоненты:

- `NewsController`;
- `NewsService`;
- `CommentService`;
- `NewsRepository`;
- `CommentRepository`;
- `News`;
- `Comment`.

Операции:

- просмотр списка новостей;
- просмотр новости;
- создание, редактирование, удаление новости;
- добавление и удаление комментария;
- постановка и снятие отметки "нравится".

## 4. Модуль расчетов менеджеров

Назначение:

- выполнение расчетов по параметрам сделки;
- подготовка отчетных данных менеджера.

Основные компоненты:

- `ManagersController`;
- `CalculationService`;
- `CalculationRequestDTO`;
- `CalculationResultDTO`;
- `InnCalculationStrategy`;
- `OooCalculationStrategy`;
- `ManagerReportService`.

Используемые данные:

- переменные из `Variable`;
- ставки НДС из `Nds`;
- коэффициенты СПК из `Spk`.

Результат работы:

- рассчитанные показатели;
- сохраненный черновик или итоговый отчет.

## 5. Модуль финансовой отчетности

Назначение:

- ведение финансовых отчетов уполномоченными сотрудниками.

Основные компоненты:

- `FinDirectorController`;
- `FinReportService`;
- `FinReportRepository`;
- `FinReport`.

Операции:

- просмотр списка отчетов;
- создание отчета;
- редактирование отчета;
- удаление отчета.

## 6. Модуль справочников

Назначение:

- централизованное хранение нормативных и справочных данных.

Основные компоненты:

- `VariableController`;
- `NdsController`;
- `SpkController`;
- `SupplierController`;
- `SoldFromCompanyController`;
- соответствующие сервисы, репозитории и модели.

Операции:

- создание записи справочника;
- редактирование записи;
- удаление записи;
- просмотр списка значений.

## 7. Модуль корпоративной базы знаний

Назначение:

- хранение и сопровождение внутренней базы знаний организации.

Основные компоненты:

- `KnowledgeBaseController`;
- `KnowledgeBasePage`;
- `KnowledgeBaseCategory`.

Операции:

- просмотр списка разделов и страниц;
- создание страницы;
- редактирование страницы;
- удаление страницы;
- создание и изменение разделов;
- загрузка изображений;
- прикрепление обложки или иллюстрации к странице.

Входные данные:

- заголовок;
- контент страницы;
- порядок сортировки;
- категория;
- изображение.

Выходные данные:

- сохраненная страница базы знаний;
- сохраненный раздел;
- JSON-ответ с URL загруженного изображения для редактора.
