# Руководство по разработке

## Как внести вклад

### 1. Сообщения об ошибках - Bugs
- Используйте GitLab Issues для сообщения об ошибках.
- Опишите шаги для воспроизведения и ожидаемый результат.

### 2. Предложение изменений - Merge Requests
1. Создайте отдельную ветку для вашей фичи или исправления ошибки.
2. Пишите понятный код с необходимыми комментариями.
3. Добавьте или обновите тесты, если это необходимо.
4. Убедитесь, что все тесты проходят успешно:
   `php artisan test` или `docker exec calcul-app php artisan test`
5. Создайте Merge Request с подробным описанием ваших изменений.
